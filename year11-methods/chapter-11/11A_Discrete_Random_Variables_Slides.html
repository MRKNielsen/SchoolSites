That's today's big idea

A random variable turns outcomes into numbers — and to know how likely each number is, you must collect the probabilities into a distribution. That's exactly what we build next.

01 · From outcomes to numbers

A random variable assigns a number to every outcome

Random variable

A random variable X assigns a number to each outcome in ε. It is discrete if it takes countable values (0, 1, 2, …) — usually from counting. (Heights and times are continuous — next year's story.)

Probability distribution

The function p(x) = Pr(X = x) — every value of X with its probability, as a rule, table or graph. Two conditions: 0 ≤ p(x) ≤ 1 and Σ p(x) = 1.

Read the diagram

Eight equally likely outcomes get sorted by their X-value. Collecting probabilities by value gives the table: Pr(X = 2) = 3/8 because three outcomes map to 2.

02 · The language of probability statements

"At least", "more than", "no more than" — get these exact

Translation table (X takes 0, 1, 2, 3, 4, 5)

"X is more than 2"X > 2: {3, 4, 5}
"X is at least 2"X ≥ 2: {2, 3, 4, 5}
"X is less than 2"X < 2: {0, 1}
"X is no more than 2"X ≤ 2: {0, 1, 2}
"between 2 and 5 (exclusive)"2 < X < 5: {3, 4}
Quick check: X takes 0–5. List the values for Pr(2 ≤ X < 5) and for "X is 2 or more".
2 ≤ X < 5: {2, 3, 4}. "2 or more" = X ≥ 2: {2, 3, 4, 5}.

The boundary is everything

"At least 2" includes 2; "more than 2" excludes it. One word, one mark. Circle the boundary value and decide: in or out?

03 · Reading a distribution

I Do Example 1

Consider the probability distribution:

  1. a Find Pr(X = 3)
  2. b Find Pr(X < 3)
  3. c Find Pr(X ≥ 4)
  4. d Find Pr(X ≠ 5)
  • aStraight from the table: Pr(X = 3) = 0.1.
  • bX < 3 means 1 or 2: 0.2 + 0.3 = 0.5.
  • cX ≥ 4 means 4, 5 or 6: 0.2 + 0.15 + 0.05 = 0.4.
  • dComplement: 1 − Pr(X = 5) = 1 − 0.15 = 0.85.

Method

Underline the values in play, add their probabilities. "Not" → complement.

04 · Valid distributions and unknowns

We Do Examples 2 and 3

Example 2 X takes values 1, 2, 3, 4. Which of these are probability distributions?

  1. a 0.05, 0.35, 0.55, 0.15
  2. b 13%, 69%, 1%, 17%
  3. c 0.66, 0.32, −0.19, 0.2
  • aSum = 1.1 ≠ 1 → not a distribution.
  • bSum = 100%, each in [0, 1] → yes — percentages are fine.
  • c−0.19 < 0 → no. A single negative entry kills it, whatever the sum.

Example 3 For what value of k is this a probability distribution?

  • 1Sum to 1: k + 2k + 3k + 4k + 5k = 1.
  • 215k = 1, so k = 115.
  • 3Check the other condition too: each entry is between 0 and 1. ✓ (Then, e.g., Pr(2 ≤ X ≤ 4) = 915 = 35.)
05 · Conditional probability from a table

I Do Example 4

For the distribution below, find Pr(X ≥ 5 | X ≥ 3).

  • 19F formula: Pr(X ≥ 5 | X ≥ 3) = Pr(X ≥ 5)Pr(X ≥ 3) — since X ≥ 5 already implies X ≥ 3.
  • 2Numerator: 0.13 + 0.23 = 0.36. Denominator: 0.07 + 0.17 + 0.13 + 0.23 = 0.6.
  • 30.360.6 = 35.

Same "given" logic as 9F

"Given X ≥ 3" restricts the world to the values 3, 4, 5, 6 — their total probability is the denominator. Nothing new: the table just replaces the Venn diagram.

Overlap check

The numerator is Pr(both conditions). Here "≥ 5 and ≥ 3" collapses to "≥ 5" — but for something like Pr(X < 3 | X < 4) you must intersect first: numerator Pr(X < 3).

08 · Summary 11A

Discrete random variables — what to remember

The objects

  • Random variable X: a number for each outcome; discrete = countable values.
  • Distribution: p(x) = Pr(X = x) as a table, rule or graph.
  • Valid ⇔ every p(x) ∈ [0, 1] and Σ p(x) = 1.

The routine

Translate the words to an inequality; circle the boundary: in or out?
Underline the values in play; add their probabilities.
Unknown k: sum to 1. Conditional: 9F formula, table-restricted.

The #1 trap

"At least" vs "more than". Pr(X ≥ 4) includes 4; Pr(X > 4) doesn't. Most lost marks in this topic are that one boundary value.
