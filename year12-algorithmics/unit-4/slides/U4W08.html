The Halting Problem

The problem

Halting Problem

Given a program \(P\) and an input \(x\): does \(P\) eventually halt when run on \(x\), or does it loop forever?

  • Last week your Turing machine simulator gave up at 2000 steps with "infinite loop?" — it couldn't know.
  • Claim (Turing, 1936): no algorithm \(H(P, x)\) can decide this correctly for all programs and inputs.
  • Not "we haven't found one yet" — provably impossible. The proof fits on one slide.

The proof (by contradiction)

  1. Suppose a halting decider \(H(P, x)\) exists: returns halts or loops, always correct, always terminates.
  2. Build a troublemaker using \(H\) as a component:
Algorithm D(P):    If H(P, P) = halts Do       # would P halt on itself?        loop forever    Else Do        Return                   # halt immediately
  1. Ask the fatal question: what does \(D(D)\) do?
  2. If \(H(D,D)\) says halts → \(D(D)\) loops forever. If \(H(D,D)\) says loops → \(D(D)\) halts. Either way \(H\) was wrong.
  3. Contradiction. No such \(H\) exists. ∎
The move to remember

Self-reference + contradiction — the same engine as "this statement is false", aimed at computation.

What undecidability means (and doesn't)

  • It answers Hilbert: the Entscheidungsproblem has no solution — mathematics cannot be fully mechanised.
  • Practical shadow: perfect malware detection, perfect infinite-loop detection, perfect program-equivalence checking — all provably impossible in general.
  • It does not mean these tools can't exist at all — it means they must be incomplete: sometimes wrong, sometimes "don't know", or restricted to special cases.
  • Contrast with intractability: NP-hard = no fast algorithm (we believe); undecidable = no algorithm at all (we've proven).

Can machines think?

The Turing Test (1950)

  • Turing replaced "can machines think?" (hopelessly vague) with an operational game: an interrogator converses by text with a human and a machine; if the machine is indistinguishable, it passes.
  • The move: define intelligence by behaviour, not by inner substance.
  • Objections Turing pre-answered: the mathematical objection (machines have Gödel/halting limits — so do humans, arguably), the consciousness objection (how would you know for a human?), Lady Lovelace's objection (machines only do what they're told).
Note the connection

The same man proved what machines cannot do (halting) and asked whether they could think. The two questions frame this whole AOS.

Strong AI vs weak AI

ClaimPassing the Turing Test would…
Weak AIMachines can simulate intelligent behaviour — act as if they understand.…demonstrate successful simulation. Fine.
Strong AIA suitably programmed machine literally has a mind — it genuinely understands.…be evidence of real understanding.
  • Weak AI is an engineering claim; strong AI is a philosophical one.
  • Searle's Chinese Room (1980) attacks strong AI specifically — next slide.

The Chinese Room

  • Setup: Searle, who knows no Chinese, sits in a room with a rulebook. Chinese questions come in; he follows the rules to assemble Chinese answers. To outsiders, the room "speaks Chinese" fluently.
  • The argument: Searle manipulates symbols by rules without understanding a word — syntax alone. If running the right program were sufficient for understanding, Searle would understand Chinese. He doesn't. So running a program is not sufficient for understanding — strong AI is false.
  • The systems reply: Searle is just the CPU; the whole system (man + rulebook + state) understands. Searle's counter: memorise the rulebook — still no understanding.
  • The robot reply: ground the symbols in sensors and action, and understanding follows. Searle: still just more symbols coming in.
