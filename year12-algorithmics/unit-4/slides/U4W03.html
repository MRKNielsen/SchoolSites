Why this matters for your SAT

If your solution contains any recursive module, Memo 02 expects its cost derived via a call graph or recurrence — VCAA names all three tools (call graphs, recurrence relations, Master Theorem) for Criterion 5–7 work.

Call graphs

A call graph shows who calls whom, how many times

Algorithm Fact(n):    If n ≤ 1 Do        Return 1    Return n * Fact(n - 1)
  • Each call does \(O(1)\) work, then makes one recursive call.
  • The call graph is a chain of depth \(n\): total \(\Theta(n)\).

A chain: depth \(n\), one call per level.

Branching recursion: the tree explodes

Algorithm Fib(n):    If n ≤ 1 Do        Return n    Return Fib(n-1) + Fib(n-2)
  • Two calls per level → a binary tree of depth \(\approx n\).
  • Call count roughly doubles per level: \(\Theta(\phi^n)\) — exponential (\(\phi \approx 1.618\)).
  • Fib(30) ≈ 2.7 million calls. Fib(50): unusable.

F(3) appears twice already — repeated work.

Key idea

Cost of recursion = (number of calls in the tree) × (work per call). The shape of the call graph is the whole story.

Halving recursion: the friendly shape

Algorithm BinSearchR(L, t, lo, hi):    If lo > hi Do        Return not_found    mid ← (lo + hi) div 2    If L[mid] = t Do        Return mid    If L[mid] < t Do        Return BinSearchR(L, t, mid+1, hi)    Else Do        Return BinSearchR(L, t, lo, mid-1)
  • One call per level, but the problem size halves: \(n \to n/2 \to n/4 \to \dots\)
  • Chain of depth \(\log_2 n\), \(O(1)\) work per call: \(\Theta(\log n)\).
  • Compare: same result as the iterative version from Week 1 — recursion doesn't change the complexity, the shape does.

Sketch the call graph of Fact(6), Fib(5), and BinSearchR on a 16-element list. Three shapes: chain, tree, halving chain.

Recurrence relations

From code to equation

  • Let \(T(n)\) = the running time on input of size \(n\). Read the recurrence straight off the code:
Code shapeRecurrenceExample
one call on \(n-1\), \(O(1)\) work\(T(n) = T(n-1) + c\)Fact
one call on \(n/2\), \(O(1)\) work\(T(n) = T(n/2) + c\)BinSearchR
two calls on \(n/2\), \(O(n)\) merge\(T(n) = 2T(n/2) + cn\)MergeSort
two calls on \(n-1\), \(O(1)\) work\(T(n) = 2T(n-1) + c\)Towers of Hanoi
  • Plus a base case: \(T(1) = c\). The recurrence is the call graph, written algebraically.

Solving by expansion (telescoping)

Example: \(T(n) = T(n/2) + c\)

\[ T(n) = T(n/2) + c = T(n/4) + 2c = T(n/8) + 3c = \dots = T(1) + kc \] where \(n/2^k = 1\), i.e. \(k = \log_2 n\). So \(T(n) = c\log_2 n + c = \Theta(\log n)\).

  • Method: substitute the recurrence into itself, spot the pattern after 2–3 steps, find when the base case is reached, sum the work.
  • Same technique gives \(T(n) = T(n-1) + c \Rightarrow \Theta(n)\), and \(T(n) = T(n-1) + cn \Rightarrow c(1 + 2 + \dots + n) = \Theta(n^2)\).
Try it!

Expand \(T(n) = 2T(n-1) + c\), \(T(1)=c\), three steps. What is the pattern, and what does it solve to?

The big one: \(T(n) = 2T(n/2) + cn\)

  • Level 0: \(cn\) work. Level 1: two problems of size \(n/2\) → \(2 \cdot c\frac{n}{2} = cn\). Level 2: four of size \(n/4\) → \(cn\). Every level costs \(cn\).
  • Depth: \(\log_2 n\) levels until size 1.
  • Total: \(cn \cdot \log_2 n + \Theta(n) = \Theta(n \log n)\) — MergeSort's bound, derived, not memorised.
The level-sum picture

Draw the call tree, write each level's total work beside it, count the levels, add. If every level costs the same, the answer is (level cost) × (depth).

The Master Theorem

The Master Theorem

For recurrences of the form

\[ T(n) = a\,T\!\left(\frac{n}{b}\right) + f(n), \qquad a \geq 1,\; b > 1 \]

Compare \(f(n)\) with \(n^{\log_b a}\) (the “tree cost”):

CaseConditionResult
1\(f(n)\) grows slower than \(n^{\log_b a}\)\(T(n) = \Theta(n^{\log_b a})\)
2\(f(n) = \Theta(n^{\log_b a})\)\(T(n) = \Theta(n^{\log_b a} \log n)\)
3\(f(n)\) grows faster (and is “regular”)\(T(n) = \Theta(f(n))\)

Intuition: case 1 — the leaves dominate; case 2 — every level ties; case 3 — the root dominates.

Worked examples

Recurrence\(a, b, f(n)\)\(n^{\log_b a}\)CaseResult
\(T(n) = 2T(n/2) + n\)\(2, 2, n\)\(n^{\log_2 2} = n\)2\(\Theta(n \log n)\)
\(T(n) = T(n/2) + 1\)\(1, 2, 1\)\(n^{\log_2 1} = 1\)2\(\Theta(\log n)\)
\(T(n) = 4T(n/2) + n\)\(4, 2, n\)\(n^{\log_2 4} = n^2\)1\(\Theta(n^2)\)
\(T(n) = 2T(n/2) + n^2\)\(2, 2, n^2\)\(n\)3\(\Theta(n^2)\)
Exam recipe

1) Read off \(a\), \(b\), \(f(n)\).  2) Compute \(n^{\log_b a}\).  3) Compare growth.  4) Quote the case and the result — name the case number in your justification.

When the Master Theorem does not apply

  • \(T(n) = T(n-1) + c\) — the subproblem shrinks by subtraction, not division (\(b \not> 1\)). Use expansion instead.
  • \(T(n) = 2T(n-1) + c\) — same problem: expansion gives \(\Theta(2^n)\).
  • Uneven splits like \(T(n) = T(n/3) + T(2n/3) + n\) — outside the standard form (level-sum picture still works).
VCAA phrasing

The study design says Master Theorem “where / if appropriate” — recognising when it applies is part of the skill. State the form check before using it.
