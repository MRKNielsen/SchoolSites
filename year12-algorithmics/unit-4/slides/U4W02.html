Key idea

Graph algorithm bounds come from one question, asked carefully: how many times is each node touched, and how many times is each edge touched?

Graph algorithm complexity

Input size for graphs: two numbers, not one

  • For lists, input size was one number \(n\). Graphs need two: \(|V|\) nodes and \(|E|\) edges.
  • They vary independently: a sparse road network has \(|E| \approx |V|\); a complete social graph has \(|E| = \dfrac{|V|(|V|-1)}{2} = \Theta(|V|^2)\).
  • A bound like \(O(|V| + |E|)\) is honest about both — collapsing it to one variable loses information.

Your SAT model: is your graph sparse or dense? What is \(|E|\) as a function of \(|V|\) in your data model?

BFS and DFS: \(O(|V| + |E|)\)

Algorithm BFS(G, start):    queue ← [start]    mark start visited    While queue not empty Do        v ← dequeue()        Foreach neighbour u of v Do            If u not visited Do                mark u visited                enqueue(u)
  • Each node enters the queue at most once (the visited check): \(O(|V|)\) dequeues.
  • When \(v\) is dequeued, we scan its neighbours: over the whole run, every edge is scanned at most twice (once from each end): \(O(|E|)\).
  • Total: \(O(|V| + |E|)\). Same argument for DFS.
Technique

Amortise over the whole run — don't multiply “neighbours per node” by “nodes”; count total edge-touches instead.

The classic trap: multiplying instead of amortising

  • Wrong: “The While loop runs \(|V|\) times and the Foreach runs up to \(|V|\) times, so BFS is \(O(|V|^2)\).”
  • That bound is not false — it's loose. The inner loop runs \(\deg(v)\) times, and \(\sum_{v} \deg(v) = 2|E|\).
  • Right: total inner-loop work across the whole run is \(O(|E|)\), so BFS is \(O(|V| + |E|)\).
When are they the same?

On a dense graph \(|E| = \Theta(|V|^2)\), so \(O(|V|+|E|) = O(|V|^2)\). On a sparse graph (\(|E| = \Theta(|V|)\)), the amortised bound is a full factor of \(|V|\) better.

Dijkstra with a binary-heap priority queue

  • Each node is extracted from the priority queue once: \(|V|\) extractions × \(O(\log |V|)\) each.
  • Each edge may trigger one distance update (decrease-key / re-insert): \(|E|\) updates × \(O(\log |V|)\) each.
  • Total: \[ O\big((|V| + |E|) \log |V|\big) \]
  • With a plain array instead of a heap: \(O(|V|^2)\) — better for dense graphs! The data structure changes the bound.

Prim's: same skeleton, same bound

  • Prim's grows a tree, always adding the cheapest edge to a new node — structurally the same loop as Dijkstra with a different key.
  • Heap version: \(O((|V| + |E|) \log |V|)\).
  • The proof-of-cost argument transfers: nodes extracted once, edges relax keys.
Exam habit

Don't just quote the bound — name the two ingredients: how many times nodes pass through the structure, and how many times edges update it.

Floyd–Warshall and PageRank

Foreach k from 1 To |V| Do    Foreach i from 1 To |V| Do        Foreach j from 1 To |V| Do            If D[i][k] + D[k][j] < D[i][j] Do                D[i][j] ← D[i][k] + D[k][j]

Three independent full loops over nodes: \(\Theta(|V|^3)\), always — best case = worst case.

  • PageRank: each iteration touches every node and edge once: \(O(|V| + |E|)\) per iteration.
  • Run for \(k\) iterations (until convergence at tolerance): \(O(k(|V| + |E|))\).
  • Note the new ingredient: an iteration count that isn't a function of graph size but of convergence behaviour.

The sentence pattern

“The loop runs ⟨count⟩ times as a function of ⟨input size⟩, the body is \(O(1)\) [or …], nesting multiplies, so the total is ⟨tightest bound⟩.”

The trap gallery

  • Two sequential loops ≠ \(n^2\): sequence adds; only nesting multiplies. \(O(n) + O(n) = O(n)\).
  • Halving/doubling loops are \(\log n\): j ← j * 2 or n ← n div 2 — count how many multiplications reach \(n\).
  • Triangular loops are still \(n^2\): \(\frac{n(n-1)}{2}\) — the ½ is a constant; don't answer \(O(n^2/2)\).
  • Early exit changes best case, not worst case: a Return inside a loop doesn't lower the worst-case bound.
  • Loose is not wrong, but loses marks: \(O(n^3)\) for a linear scan is technically true and scores poorly. Give the tightest bound.
  • Two input sizes stay two: \(O(n \cdot m)\) does not simplify to \(O(n^2)\) unless \(n = m\) is stated.

Worked example: mixed structure

Algorithm Audit(L):    seen ← 0    Foreach x in L Do                # A        seen ← seen + 1    i ← 1    While i ≤ length(L) Do            # B        Foreach x in L Do            print(i, x)        i ← i * 2
  • A: \(\Theta(n)\).   B: outer loop doubles \(i\) → \(\log n\) passes; inner loop \(n\) each → \(\Theta(n \log n)\).
  • Sequence adds: \(\Theta(n) + \Theta(n \log n) = \Theta(n \log n)\).

Worked example: definition question

Question (4 marks)

Using the formal definition of Big-O, show that \(f(n) = 3n^2 + 10n\) is \(O(n^2)\).

  • Claim constants up front: try \(c = 13\), \(n_0 = 1\).
  • For all \(n \geq 1\): \(10n \leq 10n^2\), so \(f(n) = 3n^2 + 10n \leq 3n^2 + 10n^2 = 13n^2 = c \cdot n^2\). ∎
  • Marks: choice of valid \(c\) (1), valid \(n_0\) (1), the inequality chain (2). Any valid pair works — bigger is fine.
Tip

Bound every lower-order term by the dominant term (needs \(n \ge 1\)), then add the coefficients. This always produces a valid \(c\).
