Simulated annealing

Accepting worse moves, on purpose

  • Hill climbing's flaw: it can never go downhill, so it can never leave a local optimum.
  • Simulated annealing: accept a worse neighbour with probability \(e^{-\Delta / T}\) — where \(\Delta\) is how much worse, and \(T\) is a "temperature" that starts high and cools.
  • Hot: bold exploration, almost random walk. Cold: cautious, converges like hill climbing. The schedule buys both behaviours in one run.
The metallurgy metaphor

Slowly cooled metal settles into a low-energy crystal; quenched metal freezes its defects in place. Cool the search slowly and it settles into a deep optimum instead of freezing in the first pit it finds.

Simulated annealing: the algorithm

Algorithm SimulatedAnnealing(problem, T, cooling):    current ← random solution    best ← current    While T > T_min Do        next ← random neighbour of current        Δ ← cost(next) - cost(current)        If Δ < 0 Or random(0,1) < e^(-Δ/T) Do            current ← next        If cost(current) < cost(best) Do            best ← current        T ← T * cooling            # e.g. 0.995    Return best
  • Two dials: initial \(T\) (how bold) and cooling rate (how patient). The workbook lets you feel both.
  • Keep a separate best — the walk may wander away from the best thing it ever saw.

AOS3 begins

Computer science: past and present

1900–1928: Hilbert's dream

  • David Hilbert asked for mathematics to be put on a mechanical footing — in particular the Entscheidungsproblem (1928): is there a definite procedure that decides, for any mathematical statement, whether it is provable?
  • "Definite procedure" — what we would call an algorithm — had no formal definition yet. You cannot prove no procedure exists until you pin down what a procedure is.
  • Gödel's incompleteness (1931) wounded the dream; Turing and Church (1936) finished it — by inventing the definition computer science is built on.
Key idea

Computer science was born from a question about the limits of computation — before any computer existed.

The Turing machine

  • An infinite tape of cells (the memory), a read/write head, a finite set of states, and a transition table: (state, symbol) → (write, move L/R, next state).
  • That's all. Every algorithm you have ever written can be simulated by one.
  • Church–Turing thesis: anything computable by a "definite procedure" is computable by a Turing machine. Not a theorem — a definition that has survived every challenge since 1936.

Why so deliberately simple?

  • To prove things impossible, the model must be simple enough to reason about — and powerful enough that "no Turing machine can do it" means "no algorithm can do it".
  • Universality: there is a single Turing machine that can simulate any other, given its description on the tape — the theoretical seed of the stored-program computer.
  • Next week: the payoff — a problem provably no algorithm can solve. Ever.
