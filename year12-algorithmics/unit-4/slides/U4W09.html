A different kind of algorithm

Everything until now was designed: a human specified every step. This week's algorithms are trained: the steps are fixed, but the behaviour comes from data.

Learning from data

How a data-driven algorithm learns

  • Setup: examples with labels (emails → spam/not; scans → tumour/not). Goal: a function that labels new examples correctly.
  • The loop: pick a model with adjustable parameters → measure its error on training data → adjust parameters to reduce error → repeat.
  • Learning = optimisation: minimising an error objective over parameters. You already know this landscape — it's Week 7's, with millions of dimensions.
Key idea

Training is search on an error landscape. Local optima, schedules, trade-offs — the AOS2 toolkit, aimed at parameters instead of tours.

Support vector machines: the widest street

  • Binary classification with a line (2-D) or hyperplane (higher-D) separating the classes.
  • Many lines separate the data — SVM's optimisation objective: choose the one that maximises the margin, the distance to the nearest points of each class.
  • Those nearest points are the support vectors — they alone determine the boundary.
  • Wide margin ⇒ new points near the old ones still classify correctly ⇒ better generalisation.

Not linearly separable? Add a dimension

  • Points inside vs outside a circle can't be split by any line in 2-D.
  • Create a new feature: \(z = x^2 + y^2\) (distance² from origin). In \((x, y, z)\)-space, inside-points sit low, outside-points sit high — a flat plane separates them perfectly.
  • General trick: map the data into a higher-dimensional space where a linear boundary works, then classify linearly there. (Kernels do this efficiently — the idea is what's examinable.)
Try it!

XOR-style data — (0,0), (1,1) in class A; (0,1), (1,0) in class B — is not linearly separable. Find a third feature that separates it.

Neural networks

The multi-layer perceptron (MLP)

  • Neuron: weighted sum of inputs + bias, passed through an activation function (step, sigmoid, ReLU).
  • Layers: input layer (the features) → hidden layer(s) → output layer (the classification).
  • Each connection has a weight — the parameters that training adjusts.
  • Hidden layers build intermediate features automatically — the learned version of the "add a dimension" trick.

Forward propagation: evaluating the network

Worked example (SAC-style)

Inputs \(x_1 = 1, x_2 = 0\). Hidden neuron \(h\): weights \(0.5, -0.3\), bias \(0.1\), step activation (fires if sum \(> 0\)).

\(h\): \(0.5(1) + (-0.3)(0) + 0.1 = 0.6 > 0\) → output 1.

Output neuron: weight from \(h\) is \(0.8\), bias \(-0.5\): \(0.8(1) - 0.5 = 0.3 > 0\) → classify 1.

  • That's the whole procedure: weighted sum + bias, activation, layer by layer, left to right.
  • The SAC will give you a small network and ask you to evaluate it — show the sums, not just the answers.

  • Consequence: a model is only as good as its test performance on data it never trained on. Training accuracy alone is marketing.

Ethics: when trained systems decide

  • Bias in, bias out: models trained on historical decisions inherit historical prejudice — hiring, lending, policing. The algorithm is "fair" only if the data and objective were.
  • Opacity: an MLP's decision lives in thousands of weights; explaining why one person was refused is hard. Contrast the designed algorithms of Units 3–4, which can be traced line by line.
  • Accountability: when a trained system errs, who answers — the developer, the data curator, the deployer?
  • Discussion frame for the SAC: name the stakeholder, the harm, the mechanism (data / objective / opacity), and a mitigation. Four moves, one paragraph.
