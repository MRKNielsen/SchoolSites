Key idea

The tail structure comes from the question, not the data. Decide one-tail or two-tail from the wording before you look at \(\bar{x}\).

Lesson 1 · 16C

One-tail and two-tail tests

  • Read \(H_1\) off the question's wording — decreased / increased / changed.
  • Two-tail: the one-tail probability doubles.
  • Same data can reject one-tailed yet survive two-tailed.

Choosing the alternative hypothesis

Wording of the question\(H_1\)p-value
"has the mean decreased?"\(\mu < \mu_0\)\(P(Z \leq z)\)
"has the mean increased?"\(\mu > \mu_0\)\(P(Z \geq z)\)
"has the mean changed?"\(\mu \neq \mu_0\)\(2\,P(Z \geq |z|)\)
  • Two-tail: extreme in either direction counts as evidence against \(H_0\) — so the one-tail probability doubles.
  • Same data can reject in a one-tail test yet fail to reject in a two-tail test.

Worked example: the same data, two-tailed

Example 1

Last week's drug data (\(\mu_0 = 5\), \(\sigma = 1\), \(n = 25\), \(\bar{x} = 4.6\)), but now the question is only "does the new drug change mean recovery time?"

  • \(H_0: \mu = 5\); \(H_1: \mu \neq 5\). Test statistic unchanged: \(z = -2\).
  • \(p = 2\,P(Z \geq 2) \approx 2(0.0228) = 0.0455\).
  • Still \(< 0.05\), so still reject — but only just. At \(\alpha = 0.02\) the one-tail test would reject and the two-tail wouldn't.

The same \(z = -2\), two different questions

Checkpoint 1 — your turn

Try it! (≈5 min)

A machine fills bags with \(\mu = 500\) g, \(\sigma = 8\) g. After maintenance, 16 bags average \(\bar{x} = 504.5\) g. Last week we tested "overfills". Now test: has the mean changed? Use \(\alpha = 0.05\), then \(\alpha = 0.01\).

  • \(H_1: \mu \neq 500\); \(z = 2.25\) as before; \(p = 2P(Z \geq 2.25) \approx 0.0244\).
  • At \(\alpha = 0.05\): reject — evidence the mean has changed. At \(\alpha = 0.01\): do not reject — the same data no longer convince.
  • Moral: the conclusion depends on both the tail structure and the significance level — state both before computing.
Lesson 2 · 16D

Tests and confidence intervals

  • A two-tail test at level \(\alpha\) rejects \(\mu_0\) exactly when \(\mu_0\) is outside the \((1-\alpha)\) CI.
  • Use whichever view the question asks for — and check one with the other.

Two views of one computation

Fact (16D)

A two-tail test at the 5% level rejects \(H_0: \mu = \mu_0\) exactly when \(\mu_0\) falls outside the 95% confidence interval \(\bar{x} \pm 1.96\,\sigma/\sqrt{n}\).

Example 2

A process should run at \(\mu_0 = 100\) units/hr; \(\sigma = 14\). A sample of \(n = 49\) hours gives \(\bar{x} = 104.2\). Test "has the rate changed?" at 5%, and confirm via the CI.

  • \(z = \dfrac{104.2 - 100}{14/7} = 2.1\); \(p = 2P(Z \geq 2.1) \approx 0.036 < 0.05\) → reject.
  • CI: \(104.2 \pm 1.96(2) = (100.28,\; 108.12)\) — and \(100\) lies (just) outside. Consistent. ✓

Checkpoint 2 — your turn

Try it! (≈5 min)

Same process (\(\mu_0 = 100\), \(\sigma = 14\), \(n = 49\)) but a different week gives \(\bar{x} = 103\).

  • (a) Run the two-tail test at 5%.
  • (b) Find the 95% CI and confirm it tells the same story.
  • (a) \(z = \dfrac{3}{2} = 1.5\); \(p = 2P(Z \geq 1.5) \approx 0.134 > 0.05\) → do not reject.
  • (b) CI \(= 103 \pm 3.92 = (99.08,\; 106.92)\), which contains 100 — same story, as it must.
Lesson 3 · 16E

Type I and Type II errors

  • The 2×2 table: false alarm vs miss.
  • \(P(\text{Type I}) = \alpha\) — we choose it; Type II depends on the truth.
  • Describe both errors in context, in full sentences.

Two ways to be wrong

\(H_0\) true\(H_0\) false
Reject \(H_0\)Type I errorcorrect ✓
Do not reject \(H_0\)correct ✓Type II error
  • Type I: a false alarm — concluding something changed when it didn't. \(P(\text{Type I}) = \alpha\), the significance level. We choose it.
  • Type II: a miss — failing to detect a real change. Its probability depends on the true \(\mu\), \(n\) and \(\alpha\); we don't control it directly.
  • Trade-off: shrinking \(\alpha\) makes false alarms rarer but misses more likely. More data improves both.

Errors in context

Example 3

Testing whether a new drug reduces recovery time (\(H_0\): no change):

  • Type I: approving a drug that is actually no better — patients switch for nothing.
  • Type II: shelving a drug that actually works — a real improvement is missed.
Template sentence

"Concluding that [the change happened] when in fact [it didn't]" — name the conclusion, name the truth, keep the units and the story.

  • (b) Describe a Type II error in context.
  • (c) If underfilled bags trigger fines, which error is costlier — and what would you do to \(\alpha\)?

Extension: how likely is a miss?

Example 4

Drug test again: \(H_0: \mu = 5\), \(H_1: \mu < 5\), \(\sigma = 1\), \(n = 25\), \(\alpha = 0.05\). If the drug truly gives \(\mu = 4.5\), what is the probability of a Type II error?

  • Reject when \(z \leq -1.645\), i.e. \(\bar{x} \leq 5 - 1.645(0.2) = 4.671\).
  • Type II = fail to reject although \(\mu = 4.5\): \(P(\bar{X} > 4.671)\) with \(\bar{X} \sim N(4.5,\, 0.2^2)\).
  • \(z = \dfrac{4.671 - 4.5}{0.2} = 0.855\), so \(P(\text{Type II}) = P(Z > 0.855) \approx 0.20\).
Reading

Even a genuinely better drug is missed 20% of the time with \(n = 25\). Larger samples shrink this — which is why trials are big.

Checkpoint 4 — your turn

Try it! (≈5 min)

Repeat Example 4 with \(n = 100\) (everything else unchanged: \(\mu_0 = 5\), \(\sigma = 1\), \(\alpha = 0.05\), true \(\mu = 4.5\)).

  • Now \(\text{sd}(\bar{X}) = 0.1\); reject when \(\bar{x} \leq 5 - 1.645(0.1) = 4.8355\).
  • \(P(\text{Type II}) = P(\bar{X} > 4.8355)\) with \(\bar{X} \sim N(4.5,\, 0.1^2)\): \(z = 3.355\), probability \(\approx 0.0004\).
  • Quadrupling \(n\) took the miss rate from 20% to 0.04% — sample size is the strongest lever we have.
